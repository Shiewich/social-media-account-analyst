# 脱敏与截图规则

开源仓库和公开 README 只能使用模拟数据、脱敏数据或不可反查的示例数据。真实账号分析可以本地使用，但不要提交到 GitHub。

## 不要提交

- 真实账号后台导出的 `.xlsx`、`.xls`、`.csv` 文件。
- 真实账号生成的 HTML、Markdown、JSON、PDF、长图和截图。
- 账号名、头像、主页链接、达人昵称、客户名、品牌名、IP 名、项目代号。
- 爆款或异常内容的精确标题，尤其是能被搜索引擎反查到的标题。
- 精确曝光、观看、涨粉、互动、成交、报价、投放成本等关键数据。
- 评论、私信、用户昵称、联系方式、合同、报价单或内部备注。

## 可以公开展示

- 完全模拟的样例数据。
- 已替换成通用描述的内容标题，例如“热点角色视觉实验”“虚拟品牌短片复盘”。
- 区间化或相对化的数据，例如“10w+”“高于均值 2.1 倍”“转粉效率偏高”。
- 不包含真实账号元素的示例 HTML 看板截图。

## 截图打码清单

发布图片前逐项检查：

1. 账号名、头像、账号 ID、主页链接是否已替换。
2. 总曝光、总观看、粉丝数、涨粉数、成交线索等核心数字是否已改成模拟值或区间。
3. 高低异常内容标题是否已改成泛化标题。
4. 客户、品牌、IP、合作方名称是否已泛化。
5. 文件名、表格路径、导出日期是否会暴露内部项目。
6. 评论区、私信、用户昵称是否完全移除。
7. 图片元信息或画面角落是否残留真实数据。

## 推荐流程

1. 用 `skill/scripts/sample-xhs.csv` 或自己构造的虚拟数据生成示例报告。
2. 打开 `docs/demo-interactive-report.html` 检查画面和交互。
3. 生成桌面端、移动端截图。
4. 用 `rg` 搜索真实账号名、品牌名、IP 名和爆款关键词。
5. 确认无泄露后再提交到 GitHub。
