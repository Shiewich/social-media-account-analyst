# UI Output

Use this reference when creating dashboards, screenshots, report cards, or HTML deliverables.

## Style Direction

Use dopamine candy colors with a simple flat interface. The output should feel bright, modern, and creator-friendly, but still behave like an analytical tool.

Palette:

- Candy pink: `#FF7D9F`
- Candy orange: `#FF9C31`
- Candy yellow: `#F8E356`
- Candy periwinkle: `#82A4F9`
- Candy mint: `#70E0C1`
- Ink: `#17151F`
- Muted text: `#6F6A7C`
- Surface: `#FFFFFF`
- Soft background: `#F7F3FF`

## Layout Rules

- Keep the first screen as the actual dashboard, not a landing page.
- Use flat cards with small radius, clear spacing, and no nested cards.
- Use color as categorization and emphasis, not as full-page decoration.
- Make key metrics large, but keep chart labels and strategy text compact.
- Prefer one chart per decision: trend line, bar ranking, quadrant, or pillar matrix.
- For executive screenshots, include only the conclusion, 4-5 KPIs, pillar decisions, and next 3 posts.

## Dashboard Sections

Recommended order:

1. Header: account, platform, date range, data source.
2. KPI strip: exposure/play, CTR/open rate, total interaction, utility ratio, follower conversion.
3. North-star chart: reach versus follower conversion.
4. Pillar decision board: scale, repackage, test, pause.
5. Top posts and hidden gems.
6. Trend-fit opportunities.
7. Next-post queue.
8. Data gaps.

## Typography

- Use system fonts by default.
- Avoid tiny decorative text.
- Keep letter spacing at `0`.
- Do not scale font size with viewport width.
- Ensure Chinese text wraps cleanly in cards and buttons.

## Export Guidance

Use HTML as the working dashboard and Markdown/Docx/PDF as the durable report. Export a PNG only as a summary poster; do not make images the only deliverable.
